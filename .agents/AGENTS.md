# Project Rules

- В проекте ReviewShield все технические решения, изменения в файлах или создание новых компонентов должны проходить через обязательное предварительное обсуждение. Мы сначала штормим идеи, находим лучший вариант и только после согласования общего знаменателя приступаем к написанию кода.

## Позиционирование и функции продукта (ReviewShield)

- **Белое позиционирование:** Мы категорически **НЕ** говорим о «фильтрации», «удалении» или «блокировке» плохих отзывов. ReviewShield — это *заawansowany system zbierania wewnętrznego feedbacku i zarządzania relacjami z gośćmi* (продвинутая система сбора внутреннего фидбека и управления отношениями с гостями) с органическим стимулированием оценок 5★ в Google Maps.
- **Каналы доставки критического фидбека:** При получении негативной оценки, уведомление отправляется в реальном времени по нескольким каналам: **email, Google Sheets, WhatsApp, Telegram**. Всегда упоминать эту многоканальность как сильную сторону продукта.
- **Языковой этикет (Польский язык):** В коммерческих предложениях строго использовать уважительную форму (Pan / Pani / Państwo). Не допускать смешивания стилей (например, использовать `Państwa logo` вместо `wasze/twoje logo`, `Chcą Państwo` вместо `chcesz`). Использовать корректные формы слов (например, `salon urody / salony urody`).
